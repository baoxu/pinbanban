拼板板 - 项目汇总

给合伙人看的精简版

🎯 一句话介绍

拼板板 = 滑雪板定制 + 拼单团购，让定制滑雪板从 6000 元降到 3000 多元。

类似「武士团」小程序的拼单模式，但针对滑雪板这个垂直品类做了深度定制。

💡 核心创意

痛点

  • 市售滑雪板千篇一律，不符合个人技术特点
  • 定制板太贵（8000-15000 元），普通人买不起
  • 想团购但找不到同好

解决方案

单人定制：6000 元
  ↓ 拼 1 个人
2 人拼单：4980 元/人（省 1020 元）
  ↓ 再拼 1 个人
3 人拼单：4380 元/人（省 1620 元）
  ↓ 再拼 1 个人
4 人拼单：3980 元/人（省 2020 元）

用户得实惠，工厂多订单，平台赚差价 — 三赢

指标 数据
中国滑雪人次（2025） 2800 万
滑雪装备市场规模 220 亿/年

❓ 常见问题

Q: 为什么是滑雪板？其他装备不行吗？
A: 滑雪板是滑雪装备中客单价最高、个性化需求最强的品类。板子选对了，技术进步快。先做透一个品类，再扩展到固定器、雪鞋等。

Q: 拼单失败了怎么办？
A: 有三种处理：1) 自动退款 + 优惠券补偿；2) 推荐相似拼单；3) 平台兜底成团（控制比例）。

Q: 工厂愿意配合小批量定制吗？
A: 拼单本质是「小单快反」，把多个小单合并成中单，工厂利润更高。我们已经调研过，多家工厂表示有兴趣。

Q: 和武士团有什么区别？
A: 武士团是餐饮，我们是装备。餐饮是即时消费，我们是预售定制。但拼单逻辑相通，武士团验证了模式可行性。

Q: 技术实现难度大吗？
A: 核心是拼单状态机和实时匹配，技术成熟。难点在供应链管理和用户体验，需要精细化运营。

二、数据库设计（核心表）

2.1 用户表 (users)

CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  openid VARCHAR(64) UNIQUE NOT NULL,      -- 微信 openid
  nickname VARCHAR(64),                     -- 昵称
  avatar VARCHAR(255),                      -- 头像
  phone VARCHAR(20),                        -- 手机号
  height INT,                               -- 身高 (cm)
  weight INT,                               -- 体重 (kg)
  skill_level TINYINT,                      -- 滑雪水平 1-5
  style_preference VARCHAR(32),             -- 风格偏好
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_openid (openid)
);

2.2 板型表 (board_types)

CREATE TABLE board_types (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(64) NOT NULL,                -- 板型名称
  code VARCHAR(32) UNIQUE NOT NULL,         -- 板型编码
  category TINYINT NOT NULL,                -- 类别 1=全山 2=公园 3=野雪
  difficulty TINYINT,                       -- 难度 1-5
  base_price DECIMAL(10,2),                 -- 基础价格
  description TEXT,                         -- 描述
  images JSON,                              -- 图片数组
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

2.3 拼单表 (group_orders)

CREATE TABLE group_orders (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_no VARCHAR(32) UNIQUE NOT NULL,     -- 拼单号
  board_type_id BIGINT NOT NULL,            -- 板型 ID
  board_length INT NOT NULL,                -- 长度 (cm)
  waist_width INT NOT NULL,                 -- 板腰 (mm)
  sidecut_radius DECIMAL(4,1) NOT NULL,     -- 侧切半径 (m)
  binding_angle VARCHAR(16),                -- 固定器角度
  pattern_id BIGINT,                        -- 图案 ID
  target_count INT NOT NULL DEFAULT 3,      -- 目标人数
  current_count INT NOT NULL DEFAULT 1,     -- 当前人数
  status TINYINT NOT NULL DEFAULT 1,        -- 状态 1=拼单中 2=成功 3=失败
  expire_at TIMESTAMP NOT NULL,             -- 过期时间
  created_by BIGINT NOT NULL,               -- 创建人
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_status (status),
  INDEX idx_expire (expire_at)
);

2.4 拼单成员表 (group_order_members)

CREATE TABLE group_order_members (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  group_order_id BIGINT NOT NULL,           -- 拼单 ID
  user_id BIGINT NOT NULL,                  -- 用户 ID
  role TINYINT NOT NULL DEFAULT 2,          -- 角色 1=发起人 2=参与者
  deposit_paid BOOLEAN DEFAULT false,       -- 定金已付
  deposit_amount DECIMAL(10,2),             -- 定金金额
  final_price DECIMAL(10,2),                -- 最终价格
  status TINYINT NOT NULL DEFAULT 1,        -- 状态 1=待支付 2=已支付 3=已退出
  joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY idx_group_user (group_order_id, user_id)
);

2.5 订单表 (orders)

CREATE TABLE orders (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_no VARCHAR(32) UNIQUE NOT NULL,     -- 订单号
  group_order_id BIGINT,                    -- 拼单 ID
  user_id BIGINT NOT NULL,                  -- 用户 ID
  board_type_id BIGINT NOT NULL,            -- 板型 ID
  board_length INT NOT NULL,                -- 长度
  waist_width INT NOT NULL,                 -- 板腰
  sidecut_radius DECIMAL(4,1) NOT NULL,     -- 侧切
  binding_angle VARCHAR(16),                -- 固定器角度
  pattern_id BIGINT,                        -- 图案 ID
  total_amount DECIMAL(10,2) NOT NULL,      -- 总金额
  deposit_amount DECIMAL(10,2),             -- 定金
  final_amount DECIMAL(10,2),               -- 尾款
  status TINYINT NOT NULL DEFAULT 1,        -- 状态
  factory_id BIGINT,                        -- 工厂 ID
  production_status TINYINT,                -- 生产状态
  tracking_no VARCHAR(64),                  -- 物流单号
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  paid_at TIMESTAMP NULL,
  shipped_at TIMESTAMP NULL,
  completed_at TIMESTAMP NULL,
  INDEX idx_user (user_id),
  INDEX idx_status (status)
);
