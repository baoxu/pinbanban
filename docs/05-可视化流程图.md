# 拼板板 - 可视化流程图 (Mermaid)

---

## 1. 用户拼单主流程

```mermaid
flowchart TD
    A[用户进入小程序] --> B[浏览首页/搜索]
    B --> C[选择板型开始定制]
    C --> D[Step1: 选择板型]
    D --> E[Step2: 配置参数]
    E --> F[Step3: 选择图案]
    F --> G[Step4: 价格预览]
    G --> H[创建拼单/支付定金]
    H --> I{等待拼单}
    
    I -->|拼单成功 | J[通知所有成员]
    I -->|拼单失败 | K[自动退款]
    
    J --> L[支付尾款]
    K --> M[推送优惠券]
    M --> N[推荐相似拼单]
    
    L --> O[订单进入生产]
    O --> P[生产制作 15-20 天]
    P --> Q[质检打包]
    Q --> R[发货配送]
    R --> S[用户签收]
    S --> T[评价晒图]
    T --> U[流程结束]
```

## 2. 拼单状态机

```mermaid
stateDiagram-v2
    [*] --> CREATED: 创建拼单
    CREATED --> PENDING: 支付定金
    PENDING --> PENDING: 新用户加入
    PENDING --> SUCCESS: 人数达标
    PENDING --> FAILED: 超时
    SUCCESS --> PAYING: 通知付尾款
    FAILED --> REFUNDED: 自动退款
    PAYING --> PRODUCING: 尾款付清
    PRODUCING --> QUALITY: 生产完成
    QUALITY --> SHIPPING: 质检通过
    SHIPPING --> COMPLETED: 用户签收
    COMPLETED --> REVIEWED: 评价完成
    REVIEWED --> [*]
```

## 3. 拼单匹配逻辑

```mermaid
flowchart TD
    A[用户创建拼单] --> B[提取拼单特征]
    B --> C[板型/长度/板腰/侧切]
    C --> D[检索待拼订单]
    D --> E[相似度评分]
    
    E --> F{评分结果}
    F -->|100 分 | G[完全匹配]
    F -->|80-99 分 | H[高度相似]
    F -->|60-79 分 | I[基本匹配]
    F -->|<60 分 | J[不推荐]
    
    G --> K[优先推送]
    H --> K
    I --> L[可选推荐]
    J --> M[不展示]
    
    K --> N[用户选择加入]
    L --> N
    N --> O[拼单成功]
```

## 5. 订单处理流程

```mermaid
sequenceDiagram
    participant 用户
    participant 小程序
    participant 后端
    participant 工厂
    participant 物流
    
    用户->>小程序：创建拼单
    小程序->>后端：保存订单
    后端->>后端：等待拼单
    
    loop 等待成员加入
        用户->>小程序：加入拼单
        小程序->>后端：更新进度
        后端->>小程序：推送通知
    end
    
    后端->>后端：拼单成功
    后端->>用户：通知付尾款
    用户->>后端：支付尾款
    
    后端->>工厂：发送生产单
    工厂->>后端：确认排期
    工厂->>后端：生产完成
    
    后端->>工厂：质检
    工厂->>后端：质检通过
    
    后端->>物流：发货
    物流->>用户：配送
    用户->>后端：确认收货
```
